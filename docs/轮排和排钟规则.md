基于已有的数据结构和数据库字段，重新实现轮牌队列规则和自动排钟逻辑：

# 轮钟队列规则（延续昨日轮排）

## 核心原则
- 每日开盘：继承昨日闭店时的排队顺序，只保留今日排班技师（休假除外），顺序不变。
- 休假归来、新员工、晚班到岗：追加到当前队尾（晚班同批按昨日顺序排序）。

## 单一待钟队列（队首优先接普通轮钟）

- **普通轮钟**：队首技师上钟 → 服务结束 → 回队尾。
- **点钟**：点名技师直接上钟，**其在队列中的位置保留不动**（服务结束后回到原位置）。
- **过牌（处罚）**：将该技师移出当前位置 → 追加到队尾。
- **加班**：**其在队列中的位置保留不动**（服务结束后回到原位置）。
- **休假归来**：追加到队尾。
- **早班员工**：顺序调整到队首。

# 自动排钟系统

## 目标
根据客人到店顺序、项目时长、技师性别要求，自动分配轮钟技师，同时保证：
- 不影响点钟（点名预约）。
- 不产生时间重叠。
- 尽量均匀分配，减少技师空闲时间。

## 输入
- 技师列表：含 ID、性别、当前轮牌队列顺序（基于“延续昨日轮排”规则）、当日已有的点钟及轮钟时间表（已占用时段）。
- 预约单：客人到店时间、项目时长、指定性别（男/女/不限）。

## 排钟规则

1. **性别分流**  
   若客人指定性别，则在对应性别的技师范围内独立维护一个轮牌队列（不限性别时使用全体队列）。

2. **队列基础**  
   轮牌顺序采用“先到先排”：按客人到店时间处理请求，每个技师接普通轮钟的顺序由轮牌队列决定。点钟不改变队列位置。

3. **分配算法**  
   当客人到店时：
   - 从对应性别队列的队首开始，依次检查每位技师在客人到店时间及项目时长内是否有连续空闲时段（不与已有的点钟和轮钟重叠）。
   - 找到第一个满足条件的技师，分配该预约，并将其**移入该预约时段**（服务结束后，若为轮钟则回队尾，若为点钟则保留原队列位置）。
   - 若队首技师无空闲，继续检查下一技师，直到分配成功。

4. **均匀化与减少空闲**（可选优化）  
   - 在保持轮牌顺序的前提下，允许小幅调整分配顺序：若前几位技师均不满足空闲，后续技师有空闲，则跳过前面技师直接分配该空闲技师，但被跳过的技师仍需保留下次优先权（即不改变轮牌顺序，仅因不可用而跳过）。
   - 可对技师当日已预约占用的时间做预分析，将长时间空闲的技师适当提前分配（在不违反队列顺序的前提下，仅当空档匹配时）。

5. **约束**  
   - 单技师同一时间只能有一个钟。
   - 点钟预约（点名）作为高优先级，不排队，直接插入技师时间表，且不改变技师在轮牌队列中的位置。
   - 所有分配必须满足时间不重叠。

## 输出
为每个预约单输出重排后的所有预约单，后续基于此更新或者新增预约单数据
